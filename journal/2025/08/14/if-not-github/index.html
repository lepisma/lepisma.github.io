Migrating from GitHub

personal programming

Like many others, I have been considering moving away from GitHub recently. Something happened last year too that made me consider this. At that time, I started a self-hosted forgejo that became my primary store with GitHub syncing set up for select few repositories. But I soon realized that my projects directly syncing across machines via a self-hosted Nextcloud instance was all I needed. Git or no git, I am good to seamlessly switch machines and resume working. So, back to GitHub. In some ways, me migrating from GitHub will not cause any meaningful impact on the larger system. But I believe in the value of symbolic acts that could have higher order effects.

So this time, I am trying to dissect my usage of GitHub and then decide how to migrate and where to. Here is what I use my GitHub account for, other than hosting my own repositories:

  1. To show my programming portfolio
  2. As identity provider for various third party tools
  3. Hosting static pages for this website and other projects
  4. Accessing other projects and organizations that are on GitHub

The repositories themselves lie in the following categories:

  1. Personal repositories and projects that are just up for display, like my Emacs configuration, various experiments, and prototypes. I do not expect anyone else to collaborate on these.
  2. Projects that are exposed for others to use. There might be social interactions here in form of pull-requests, issues, code reviews, questions, etc. Repositories like mbzlists are good examples of this kind.
  3. Projects that are built collaboratively where the team has decided to use GitHub for working. Many of these are private.

As a rough guess, the first amount to ~70% of my repositories, second around 10%, and rest of the 20% cover third category at the moment. Additionally there are private repositories in my local machine that are not on GitHub at all.

And here are things that I don't use GitHub for:

  1. I do not use the GitHub home page feed which just seems to be a distraction.
  2. I don't want to use and think about stars in any shape or form.
  3. I don't use GitHub actions that much. Most of my actions are for hosting static pages.
  4. I do not use GitHub copilot or any other AI features.
  5. I do not use their project management features other than basic issues.

Based on all this, I think the following seems reasonable to do:

  1. Upgrade my project page to display a custom portfolio of projects by crawling my local projects directory. There are static site generators like itsy-gitsy that I might take inspirations from. I might end up showing the current tree, README, etc. from the repository while putting the whole project in a tarball for download or redirecting to a forge like Codeberg wherever it makes sense.
  2. Set up a Codeberg account for repositories of category 2 and, if possible, 3. I also thought about sourcehut but I feel the familiarity of gitea, forgejo, GitHub, style interface would make repositories of this kind more accessible. In any case, since my main entry point for projects of any kind will be my project page on this website, I can totally see myself using a mix of GitLab, Codeberg, sourcehut, etc.
  3. Archive all GitHub projects and put redirect notice on them, and my profile. There are old links that I want to preserve.
  4. Migrate GitHub identity usage to other options, as much as I can.
  5. Move static page hosting to netlify, Neocities, or other alternatives.
  6. Set up CI/CD on my local server for any post commit workflow that I might have missed noticing.

I still see my GitHub account to be active for collaborating on projects that choose to stay on GitHub. But I will move most of my activities out.

Other than the logistical aspects, I am excited about the idea of building a custom projects page that breaks out of the restrictions of how a forge like GitHub shows a user's profile. I am hoping to have a version over the next few weeks.
